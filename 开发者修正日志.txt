版本v1.3（2026.07.01）
数据库与前端数据层可信化修正：

1. 数据库扩展 ✅7.01
   o fetch_data.py 从单一 A 股指数池升级为双数据源资产池：
     - AkShare：上证指数、深证成指、创业板指、沪深300、沪深300(深)、上证50、中证500、中证1000、科创50、中证800、中证全指、中证红利。
     - yfinance：纳斯达克综合ETF(ONEQ)、黄金ETF(GLD)、半导体ETF(SOXX)、医药健康ETF(XLV)。
   o 每个 data/*.json 保持前端兼容格式：dates + values，其中 values[i][1] 仍为 close，不破坏现有模型算法读取方式。
   o 每个 JSON 新增 meta 字段，记录中英文名称、资产类别、区域、风格、频率、数据源、更新时间。

2. 数据质量与维护 ✅7.01
   o 新增 data/asset_manifest.json，作为前端统一资产清单。
   o 新增 data/quality_report.json，记录每个资产的抓取状态、起止日期、行数、失败原因。
   o GitHub Actions update_data.yml 增加 workflow_dispatch、contents: write、concurrency；支持每日自动更新和手动触发。
   o 新增 stale cache 保护：当 yfinance 等外部源日内间歇失败时，脚本保留仓库中已有 JSON，不让资产从 manifest 消失；quality_report 中以 stale_cache 标记。

3. 两个模型接入 manifest ✅7.01
   o backtest.html 与 markowitz.html 改为优先读取 data/asset_manifest.json 自动生成资产池、输入联想、基准选择。
   o manifest 读取失败时保留原有硬编码资产池作为 fallback，保证页面不因数据源异常而空白。
   o 数据 URL 改为相对路径 data/...，GitHub Pages 与本地静态服务器均可运行。
   o 未改动 runBacktest、solveQP、monteCarloOptimize、runFullAnalysis 等核心算法逻辑。

4. Data Health 与快速冷启动 ✅7.01
   o 两个页面新增 Data Health 状态条，展示数据状态、可用资产数、最新日期、manifest 日期。
   o manifest 与行情数据请求超时从 1.5 秒收紧到 0.5 秒。
   o 页面初始化改为先使用内置资产池/模拟缓存快速渲染初始图表，再异步加载真实 manifest 和真实数据；网络慢或数据源不可达时，0.5 秒内降级到 cachedData。
   o 新增浏览器 localStorage 缓存层：manifest 与每个资产 JSON 成功读取后会写入本地浏览器缓存；后续网络超时或服务器短暂不可达时，0.5 秒内优先读取浏览器缓存，再降级到内置模拟数据。
